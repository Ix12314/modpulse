# 热门模组网站 PRD（产品需求文档）

## 1. 产品概述

一个展示 Minecraft 实时热门模组的网站，数据来源于 Modrinth 公共 API（无需鉴权，支持 CORS）。
- 解决用户「快速浏览当前最热门模组、查看模组详情」的需求
- 目标用户：Minecraft 玩家、模组爱好者、服务器管理员
- 市场价值：聚合实时模组热度数据，提供 Apple 风格的优雅浏览体验

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 访客 | 无需注册 | 浏览热门模组、搜索、查看详情、切换主题 |

### 2.2 功能模块

1. **首页**：Hero 区、实时热门模组榜单、分类筛选、搜索
2. **模组详情页**：模组信息、描述、统计图表、版本列表、下载链接

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 首页 | Hero 区 | 大标题、副标题、实时数据计数器、搜索框 |
| 首页 | 热门模组榜单 | 实时下载量排名、分类标签、卡片网格 |
| 首页 | 分类筛选 | 按模组类型/加载器/版本筛选（fabric/forge/quilt） |
| 首页 | 排序控制 | 按下载量/关注数/最新更新排序 |
| 模组详情页 | 基础信息 | 图标、标题、作者、描述、统计 |
| 模组详情页 | 统计卡片 | 下载量、关注数、最新版本、更新时间 |
| 模组详情页 | 版本列表 | 支持的游戏版本、加载器、下载按钮 |
| 全局 | 主题切换 | 深色/浅色模式切换，跟随系统偏好 |

## 3. 核心流程

用户打开网站 → 系统实时拉取 Modrinth API 热门模组数据 → 渲染卡片网格 → 用户可搜索/筛选/排序 → 点击卡片进入详情页 → 详情页展示完整模组信息与统计

## 4. 用户界面设计

### 4.1 设计风格

- **主色**：浅色模式 `#fbfbfd` 背景 + `#0071e3`（Apple Blue）强调色；深色模式 `#000000` 背景 + `#0a84ff` 强调色
- **次色**：中性灰阶 `#1d1d1f` / `#f5f5f7`，强调色辅以 `#34c759`（成功）、`#ff9500`（警告）
- **按钮风格**：胶囊形（pill）、圆角 980px、轻微投影、hover 时微缩放
- **字体**：SF Pro 系统字体栈（-apple-system, BlinkMacSystemFont, "SF Pro Display", "SF Pro Text"），标题超粗（700-800），正文常规（400）
- **布局**：卡片网格（自适应列数）、顶部固定导航栏、最大宽度 1280px 居中
- **图标**：lucide-react 线性图标，1.5px 描边，与 Apple HIG 一致
- **特效**：毛玻璃（backdrop-blur）导航栏、卡片 hover 上浮 + 投影增强、骨架屏加载态

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|----------|
| 首页 | 顶部导航栏 | 毛玻璃背景、Logo、导航链接、主题切换按钮 |
| 首页 | Hero 区 | 超大标题、渐变背景、实时模组计数器、搜索框 |
| 首页 | 筛选栏 | 胶囊按钮组（加载器/类型）、排序下拉、结果计数 |
| 首页 | 模组卡片 | 图标、标题、作者、描述截断、下载量徽章、分类标签 |
| 模组详情页 | 头部 | 大图标、标题、作者、描述、操作按钮 |
| 模组详情页 | 统计区 | 4 列统计卡片（下载/关注/版本/更新时间） |
| 模组详情页 | 版本表 | 表格：版本号、游戏版本、加载器、下载按钮 |

### 4.3 响应式

- 桌端优先（≥1024px）：4 列卡片网格
- 平板（768-1023px）：2-3 列
- 移动（<768px）：1 列，导航折叠为汉堡菜单，触控优化

### 4.4 3D 场景

本项目不涉及 3D 场景。

## 5. 数据来源

- **Modrinth API**（公共、免鉴权、CORS 启用）：
  - `GET /v2/search` - 搜索/排序模组（按 downloads/follows/updated）
  - `GET /v2/project/{id|slug}` - 获取模组详情
  - `GET /v2/project/{id|slug}/version` - 获取版本列表
  - `GET /v2/tag/category` - 获取分类标签
- 数据为实时拉取，无本地缓存
